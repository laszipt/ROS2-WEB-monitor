BORS Web Dashboard

Monitoring and control interface for ROS systems

Overview

The BORS Web Dashboard provides a modern, browser-based interface for monitoring and controlling ROS (Robot Operating System) applications. It uses WebSockets to establish a real-time connection between the browser and ROS, making it easy to visualize topics, monitor system performance, and interact with your ROS environment.

Features

  • Real-time monitoring of ROS topics
  • System resource visualization (CPU, memory, disk)
  • Node and topic listings
  • Message logging
  • Responsive design works on desktops, tablets, and mobile devices

Prerequisites

  • ROS 2 environment
  • Python 3.6+
  • Modern web browser

Installation

Install the required Python packages:

pip install websockets psutil

Usage

Starting the WebSocket bridge

Run the WebSocket bridge to connect your ROS environment to the web interface:

python3 ros_ws_bridge.py

Optional command line arguments:

  • --host: Host to bind to (default: 0.0.0.0)
  • --port: Port to bind to (default: 9090)
  • --ros-node-name: ROS node name (default: ros_ws_bridge)

Accessing the dashboard

Once the WebSocket bridge is running, open the dashboard in your web browser:

http://localhost:9090

Or if accessing from another computer:

http://[server-ip-address]:9090

Dashboard Components

Navigation

The navigation bar at the top provides quick access to different sections of the dashboard and displays the connection status to the WebSocket server.

System Status

Monitor system resources and overall status at a glance, including:

  • ROS connection status
  • System operational status
  • Active node count

Performance Charts

Visualize system performance metrics with real-time charts:

  • CPU and memory usage over time
  • Resource allocation breakdown

Topics and Messages

View active topics and their messages:

  • List of all available topics with types and frequencies
  • Real-time message log for subscribed topics

Customization

The dashboard is built with Bulma CSS framework, making it easy to customize:

  • Modify index.html to change the layout and components
  • Edit js/main.js to adjust functionality and data handling
  • Customize ros_ws_bridge.py to modify the WebSocket server behavior

JavaScript API

The dashboard exposes a JavaScript API for custom integrations: 

// Subscribe to a ROS topic
BORS.subscribeToTopic('/my_topic', 'std_msgs/String');

// Listen for messages from a specific topic
BORS.addMessageListener('topic:/my_topic', (message) => {
    console.log('Received message:', message);
});

// Send a custom command to the WebSocket server
BORS.sendMessage('custom_command', { param: 'value' });

Troubleshooting

Connection Issues

If the dashboard shows "Disconnected":

  • Verify that the WebSocket server is running
  • Check if the correct WebSocket URL is configured in the browser
  • Ensure no firewall is blocking the WebSocket port

Missing ROS Topics

If expected ROS topics are not showing up:

  • Verify that your ROS environment is properly sourced
  • Check if the topics are actually active in ROS (using ros2 topic list)
